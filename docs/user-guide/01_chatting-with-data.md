# Chatting with Data

This guide explains how to use the chat interface to query your database with natural language.

---

## The Chat Interface

Navigate to **Chat** in the top navigation. The page has two areas:

| Area | Description |
|---|---|
| **Left sidebar** | Schema Explorer — browse schemas, tables, and columns |
| **Main area** | Message thread + input bar |

### Toolbar (top of main area)

The toolbar is split into two sides:

**Left side**
- **Provider selector** — choose which LLM to use for this session
- **Execution mode selector** — set the current mode (Auto-Execute / Review First / Generate Only)

**Right side**
- **Share Session** — copies a shareable link to the current session (appears once the chat has messages)
- **Token usage badge** — running total of input/output tokens consumed this session (appears once tokens are used)
- **New Chat** — start a fresh session on the current connection
- **Workspace toggle** — opens/closes the workspace panel; shows the active connection name and source type (e.g. `my_db · postgresql`)

The active connection is changed from the **Workspace panel**, not the toolbar.

---

## Sending a Question

Type your question in the input bar at the bottom and press **Enter** (or click the send button).

### Good questions to ask

```
How many customers signed up this month?
Show me the top 10 orders by total amount
Which products have never been ordered?
What is the average order value by customer country?
List customers who placed more than 5 orders
Show sales revenue grouped by month for the last year
Which categories have the fewest products?
```

### Tips for better results

- **Be specific about time ranges:** "last 30 days" is clear; "recent" is ambiguous
- **Name tables or columns if you know them:** "Orders from the orders table where status is 'completed'"
- **Use follow-up questions:** The chat is multi-turn — you can refine within a session
- **Generate the semantic model first:** If column names are cryptic, generate the semantic model to give the LLM business context

## Understanding the Response

Each assistant response contains:

### 1. Identity Row

Shows `savvina ai` and, once the query executes, the execution time in milliseconds (e.g. `savvina ai · 142ms`).

### 2. Cache Badge (if applicable)

If the query was served from cache, a **✓ Cached** badge appears here. See [Cache Hits](#cache-hits) below.

### 3. Generated Query

The SQL query generated by the LLM, displayed with syntax highlighting. The query has been:
- Validated as read-only (SELECT / WITH only)
- Checked for dangerous functions and patterns
- Automatically given a LIMIT clause if none was present

The card header shows the SQL dialect badge, the row count, a per-message token usage indicator, and a **Copy** button.

### 4. Results (if executed)

A results card with a tab bar. See [Working with Results](#working-with-results) below.

### 5. Explanation

A brief description of what the query does and why the LLM structured it that way, shown below the results card. Includes a **Copy** button and feedback actions.

### 6. Error Banner

If query generation fails or the query cannot execute, a red error banner shows the error message. If schema data was unavailable, a yellow info banner is shown instead.

## Working with Results

The results card has a tab bar across the top, with action buttons on the right.

### Tabs

| Tab | Content |
|---|---|
| **Table** | Paginated data table with sortable columns |
| **Stats** | Per-column statistics (min, max, distinct count, null count, etc.) |
| **Chart (named)** | A chart you have created; up to 5 charts per result set |

Click **+ New Chart** in the tab bar to add a chart tab. Charts can be deleted from the dropdown (▾) on their tab.

### Table view details

- Column headers are clickable to **sort** ascending/descending
- For truncated result sets, sorting re-executes the query with `ORDER BY` against the database (shown as "sorted by db" in the footer)
- NULL values are shown as italic _NULL_
- Long cell values truncate with a tooltip showing the full value
- Footer (left): row count, `· truncated` if the result was capped, `· sorted by db` if a server-side sort was applied
- Footer (right): bytes scanned (where reported by the data source)

### Action buttons (right side of tab bar)

| Button | Action |
|---|---|
| **CSV** | Download results as a CSV file |
| **Excel** | Download results as an Excel file |
| **PNG** | Export the active chart as a PNG image (chart tabs only) |
| **SVG** | Export the active chart as an SVG file (chart tabs only) |
| **Settings** | Toggle the chart settings panel to configure axes, aggregation, etc. (chart tabs only) |
| **Share** | Copy a shareable link for this result set to the clipboard |
| **Expand / Minimize** | Toggle full-screen view of the results card (Esc also closes) |

## Cache Hits

When you ask a question that is semantically similar to a previously asked question, Savvina AI returns the cached query instantly — no LLM call is made. The response is marked with a **✓ Cached** badge.

The cache uses two levels:
1. **Exact match** — normalized (lowercase, trimmed) question text matches exactly
2. **Semantic match** — cosine similarity of question embeddings exceeds the threshold (default 0.87)

### Refreshing a cached result

A **Refresh** button appears next to the **✓ Cached** badge. Clicking it bypasses the cache and asks the LLM to regenerate the query, then re-executes it. Use this if the underlying data has changed or you want a fresh query.

## Feedback Buttons

Every successful assistant response shows two feedback buttons next to the explanation.

| Button | What it does | Side-effect |
|---|---|---|
| **👍 Thumbs up** | Marks this answer as correct | Saves the question → SQL pair to the **Example Library**. Future semantically similar questions will include this pair as a few-shot hint, improving generation accuracy over time. |
| **👎 Thumbs down** | Marks this answer as wrong | Evicts this question and semantically similar entries from the **query cache**. The next time the same (or similar) question is asked the LLM regenerates the query from scratch instead of reusing a cached result. |

> **Note:** Thumbs down clears the cache but does not prevent the LLM from generating the same query again. If the query is structurally wrong, use the **Edit SQL** button to correct it, or refine your question.

### Undoing feedback

Click the active (highlighted) button again to retract your rating:

- Retracting a 👍 removes the entry from the Example Library so it will no longer influence prompts.
- Retracting a 👎 clears the rating field. Cache entries that were already evicted are not restored, but the feedback slot is freed so you can re-rate the message.

While one rating is active, the opposite button is disabled — retract the current rating first if you want to switch.

## Multi-Turn Conversations

Savvina AI maintains conversation history within a session. You can ask follow-up questions:

```
You:       Show me the top 5 customers by total spend
Assistant: [results]

You:       Now filter that to only customers in Germany
Assistant: [refined results]

You:       Add a column showing their average order value
Assistant: [further refined results]
```

The **20 most recent turns** of the session are sent to the LLM as conversation context on each message. Earlier turns in very long sessions are not included.

## Reviewing and Editing Queries (Review First Mode)

If the connection uses **Review First** mode, each generated query appears in a review panel before execution:

```
┌─────────────────────────────────────────────────┐
│ 📝 Generated Query                    [Edit ✏️]  │
│ ┌─────────────────────────────────────────────┐ │
│ │ SELECT c.name, SUM(o.total) AS revenue      │ │
│ │ FROM customers c                            │ │
│ │ JOIN orders o ON c.id = o.customer_id       │ │
│ │ WHERE o.status = 'completed'                │ │
│ │ GROUP BY c.name                             │ │
│ │ ORDER BY revenue DESC LIMIT 10              │ │
│ └─────────────────────────────────────────────┘ │
│   [▶ Run Query]  [✏️ Edit]  [📋 Copy]  [Cancel]  │
└─────────────────────────────────────────────────┘
```

| Button | Action |
|---|---|
| **Run Query** | Execute as-is |
| **Edit** | Opens the query in an editable textarea; then "Run Edited Query" executes your modified version |
| **Copy** | Copy to clipboard; paste in your own SQL client |
| **Cancel** | Discard this query |

Edited queries go through the same read-only safety validation before execution.

## Chat History

Click **History** in the navigation to see all past chat sessions. Each session shows:
- Session title (auto-generated from the first message)
- Connection used
- LLM provider
- Last active timestamp
- Number of cache hits in that session (shown as ⚡ N cached)

Click any session to resume it and continue the conversation.

## Schema Explorer

The left sidebar shows the database schema as a collapsible tree:

```
▼ public
  ▼ customers  (TABLE · ~12,450 rows)
      id           uuid  [PK]
      email        text
      first_name   text
      last_name    text
      created_at   timestamp
  ▼ orders  (TABLE · ~89,230 rows)
      id           uuid  [PK]
      customer_id  uuid  [FK → customers.id]
      total        decimal
      status       text
```

- Row counts are **approximate** (prefixed with `~`)
- Click any **table or column name** to copy it to the clipboard
- If a **semantic model** exists, the display name appears alongside the raw name: `tbl_cx_mstr (Customers)`
- Use the **search box** at the top to filter tables and columns
